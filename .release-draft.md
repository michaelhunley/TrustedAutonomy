## v0.13.1-alpha.7

### New Features

- **Autonomous Operations & Self-Healing Daemon** — The daemon now monitors its own health and logs corrective actions. `ta operations log` surfaces stuck goals, disk pressure warnings, and other operational issues with severity levels (Info / Warning / Critical) and suggested fixes. The daemon emits low-disk alerts when available space drops below 2 GB.
- **Intelligent Status Dashboard** — Running `ta` with no arguments now shows a prioritized dashboard: urgent items (stuck/failed goals, pending drafts), active agents, recent completions, and suggested next actions. Previously this required `ta status`.
- **Operational Runbooks** — `ta runbook list/show/run` provides five built-in runbooks (disk-pressure, zombie-goals, crashed-plugins, stale-drafts, failed-ci) with dry-run preview and an approval flow before any destructive action.
- **Natural-Language Intent Routing in Shell** — Phrases like "what's stuck?" or "clean up old goals" in the shell now map to concrete `ta` commands rather than being sent to the agent as freeform text.
- **Game Engine Project Templates** — `ta init --template unreal-cpp` and `ta init --template unity-csharp` generate a full project scaffold including BMAD agent config, Claude Flow `.mcp.json`, a discovery goal, game-engine `.taignore` / `policy.yaml`, and seeded memory entries for Unreal Engine 5 and Unity conventions.
- **Power & Sleep Management** — The daemon now prevents macOS/Linux system sleep while a goal is actively running (using `caffeinate -i -s` on macOS and `systemd-inhibit` on Linux). Sleep detection via wall-vs-monotonic clock divergence triggers a `SystemWoke` event and a connectivity re-check after wake. `ta status --deep` reports whether a sleep assertion is active.
- **Daemon Auto-Install** — `ta daemon install` generates a macOS LaunchAgent plist or Linux systemd user-service unit so the daemon starts on login. Run without `--apply` to preview the generated unit before installing.
- **Lifecycle Compaction** — `ta gc --compact` removes staging directories and draft packages for goals in Applied/Completed state older than a configurable threshold (`daemon.toml` `[lifecycle.compaction]`), recovering disk space without losing history entries.
- **Reflink / COW Overlay** — Staging copies now use APFS `clonefile(2)` on macOS and `FICLONE` ioctl on Btrfs/Linux where available, making goal start near-instant on supported filesystems. Falls back to a full copy automatically.
- **Follow-Up Draft Chain Context** — Follow-up drafts now carry the parent goal's title. `ta draft view` shows "follow-up to \<title\>" and lists the chain with file counts; `ta draft apply` prints a "Changes from parent:" rollup when applying a root draft that has children.
- **Release Label Dispatch** — `ta release run <version> --label <tag>` runs the normal release pipeline and then automatically dispatches the GitHub Actions workflow with a public-facing label tag (e.g. `public-alpha-v0.13.1.7`). The `--prerelease` flag marks the dispatched release as a pre-release.
- **Release History Tracking** — Each successful `ta release run` now writes a record to `.ta/release-history.json` (version, tag, commit SHA, timestamp). The next release reads this file first to determine the exact commit delta, eliminating dependence on `git describe` and bookkeeping tags.

### Improvements

- **Shell `run` Shortcut** — `run <title>` is now a first-class shell alias for `goal <title>`, matching how users naturally phrase requests.
- **`vcs` Command Alias** — Shell commands previously typed as `git <cmd>` can now be entered as `vcs <cmd>` (the canonical form), with `git` kept as a backwards-compatible alias.
- **Keybinding Help from Single Source of Truth** — Shell keybindings are now defined in `KEYBINDING_TABLE`; the help text and the dispatch table are generated from the same constant, preventing documentation drift.
- **`!<cmd>` Shell Escape Documented** — The `!<command>` passthrough for running arbitrary shell commands is now listed in the shell's help output and in USAGE.md.
- **Notifications API** — New `GET /api/notifications` endpoint returns urgent items (stuck/failed goals, pending drafts) with urgency sorting and stable dedup IDs, enabling external tools and the web shell to surface actionable alerts.
- **Apply Pipeline Error Messages** — When `ta draft apply` fails, the output now includes the number of rolled-back files, the root cause, and three numbered retry options with the exact command to re-run.
- **Release Pipeline Error Messages** — Failed release pipeline agent steps now include the goal title (not just the UUID), plain-English failure context, and numbered recovery steps.
- **`--from-tag` Override for Release Notes** — `ta release run <version> --from-tag <tag>` explicitly pins the base tag for commit-range computation, bypassing `git describe` for cases where bookkeeping tags are missing.
- **Daemon Build Included in Release Artifacts** — All five release platform packages now include the `ta-daemon` binary alongside `ta`. Previously only `ta-cli` was shipped, making every fresh install non-functional for `ta shell`, `ta run`, and `ta daemon start`.
- **Windows Daemon Discovery Fixed** — `find_daemon_binary()` now uses `EXE_SUFFIX` for sibling lookup and `where` instead of `which` on Windows, taking only the first line of output.

### Bug Fixes

- **Apply Pipeline Ordering (Bug D)** — `PLAN.md` was being written before `git checkout -b`, leaving a dirty working tree that git refused to check out from. The plan-update step now runs inside the VCS adapter's commit closure, after `prepare()` and before `commit()`.
- **Working Indicator Lingering After Agent Exit (R1)** — The "Agent is working…" line occasionally remained visible after a goal completed. Fixed by ensuring `AgentOutputDone` scans and clears all heartbeat lines (not just the most recent one) in both the main and agent output panes.
- **Auto-Scroll Not Firing on All Output Paths (R2)** — `SseEvent`, `CommandResponse`, `DaemonDown`, and `DaemonUp` message handlers were missing `auto_scroll_if_near_bottom()` calls, leaving the view stale when near (but not at) the tail. All four paths now scroll correctly.
- **Discord Duplicate Notifications** — `handle_draft_ready` was firing on both `draft_built` (internal) and `review_requested` events, sending 2–3 duplicate Discord messages per goal. It now fires only on `review_requested`.
- **Discord "(unnamed goal)" in Notifications** — The `ReviewRequested` event now carries the goal title, so Discord notifications show the actual goal name.
- **Discord Approve/Deny Buttons Silently Dropped** — The Discord listener was only handling interactions with a `ta_` prefix; `draft_<uuid>_approve|deny` button IDs were silently ignored. The handler now routes these directly to `POST /api/drafts/<id>/approve|deny`.
- **Case-Insensitive `ta` Command Prefix** — Commands like `Ta run goal` or `TA run goal` now route correctly. Fixed in the shell, Discord listener, and daemon router.
- **`FICLONE` ioctl Type Mismatch on `aarch64-linux-musl`** — The ioctl request type was `u32` on some targets; cast to `libc::Ioctl` to fix a compile error on aarch64 musl.
- **ETXTBSY Race in VCS Plugin Tests** — Added a retry loop for `ETXTBSY` errors when spawning VCS plugins on overlayfs-backed `TMPDIR` (Nix CI), where the kernel copy-up races against binary execution.
