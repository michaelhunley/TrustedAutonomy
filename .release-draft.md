## TA v0.14.22 — TA Studio + Governed Workflow Phase Targeting

This release delivers the TA Studio UI milestone and reliable plan-phase automation.

---

### TA Studio

A native desktop UI for managing goals, plans, and drafts without the CLI.

- **Multi-project browser** — switch between projects from the sidebar; each gets its own daemon connection
- **Plan tab** — browse PLAN.md phases, run a phase with one click, or enter a custom goal; phase status updates live
- **Workflows tab** — trigger and monitor governed workflow runs from the UI
- **Agent Personas** — configure per-project agent identity (name, model, system prompt) from a dedicated tab
- **New Project Wizard** — guided setup flow for new TA projects; guard prevents re-running in an already-initialised project
- **Platform launchers** — `ta-studio` binary + `.app` / `.bat` launchers included in release packages

### Governed Workflow Phase Targeting

`ta workflow run build --goal "v0.4.0 — Title" --phase v0.4.0`

- `--phase` flag threads the plan phase ID through the entire governed workflow:
  - Injected into the agent's CLAUDE.md context via `ta run --phase` so the agent focuses on the right phase
  - Passed to `ta draft apply --phase` so PLAN.md is marked done after apply
- **Empty-draft guard** — if the agent produces no changes, the workflow now fails immediately with a clear actionable message instead of silently completing all stages and looping on the same phase forever
- `build_phases.sh` updated to pass the full goal (`"$PHASE_ID -- $PHASE_TITLE"`) and `--phase`

### Unified Project Init

`ta init` and `ta new` merged into a single consistent flow. `ta plan new` scaffolds a PLAN.md with a starter roadmap.

### Release Packaging

- Windows MSI now correctly bundles `ta-studio.bat` (fixed WIX0103 build error)
- `USAGE.html` included in all platform packages (macOS DMG, Windows MSI, Linux tarball)
