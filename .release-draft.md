## v0.6.3-alpha

### New Features

- **Active Memory Injection** — Agents launched via `ta run` now receive structured architectural knowledge, conventions, and project state from the memory store — eliminating hours of codebase re-discovery at session start. Context is phase-filtered and category-prioritized so agents see only what's relevant.
- **Project-Aware Key Schema** — Memory keys auto-detect project type (Rust workspace, TypeScript, Python, Go) and use domain-specific naming (`arch:crate-map` vs `arch:package-map`). Configurable via `.ta/memory.toml` or auto-detected from filesystem signals.
- **Session & Human Control Plane** — New `ta-session` crate with continuous conversation model between human and TA. `TaSession` lifecycle (Starting → AgentRunning → DraftReady → WaitingForReview → Iterating → Completed), `SessionManager` with persistent storage, and CLI commands: `ta session pause`, `ta session abort`, `ta session status`.
- **Unified Policy Config** — All supervision configuration resolves to a single `PolicyDocument` loaded from `.ta/policy.yaml`. Six-layer tighten-only cascade: built-in defaults → project → workflow → agent → constitution → CLI overrides. Security levels (Open < Checkpoint < Supervised < Strict) can only tighten as layers merge.
- **Resource Mediation Trait** — New `ta-mediation` crate generalizes the file staging pattern to any resource type. `ResourceMediator` trait with stage/preview/apply/rollback lifecycle, `FsMediator` for `fs://` URIs, and `MediatorRegistry` for scheme-based routing.
- **Negative Path Tracking** — Rejected draft approaches are stored as `NegativePath` memory entries so agents avoid repeating known-failed strategies. Keyed by phase and rejection slug.
- **`ta context schema` command** — Inspect the auto-detected key schema for your project, including project type, memory backend, domain mapping, and special key patterns.

### Improvements

- **RuVector default-on** — Semantic vector search is now enabled by default. Agents get better relevance ranking in memory queries without extra configuration.
- **Architectural knowledge extraction** — Goal completion events automatically extract module maps from change summaries, giving future agents instant understanding of project structure.
- **Runtime-aware policy evaluation** — New `evaluate_with_document()` on PolicyEngine layers document-level checks: scheme approval, agent overrides, drift escalation, action count limits, budget limits, and supervised mode enforcement.
- **Budget tracking** — `BudgetConfig` with `max_tokens_per_goal` and `warn_at_percent` (default 80%) for per-goal token budget monitoring.
- **Product Concept Model** — ADR codifying TA's 5-layer architecture (Mediation → Policy → Session → Memory → Channels) with extension points documentation and paid add-on specs.
- **Restructured roadmap** — v0.6+ plan phases aligned to the 5-layer architecture model. Virtual Office and Infra Ops moved to separate project tracks.

### Bug Fixes

- **Stale sub-goal injection removed** — Cleaned up leftover v0.5.7 sub-goal injection from CLAUDE.md that could confuse agent context on session launch.
- **Invisible TA design refinement** — Fixed session flow so TA remains fully transparent to agent frameworks — agents work, exit, and TA captures the result without framework coupling.
