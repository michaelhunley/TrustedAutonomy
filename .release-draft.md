## v0.12.8-alpha

This release covers all changes since v0.10.12-alpha, spanning the v0.10.13–v0.12.8 development cycle.

### New Features

- **`ta new` — Conversational Project Bootstrapping** — Start a new project with a guided agent conversation. `ta new` initialises a `.ta/` workspace, creates a `PLAN.md`, and launches an interactive session to configure the project. Replaces manual workspace setup.

- **`ta plan add` — Agent-Powered Plan Updates** — Add new phases, items, or sections to PLAN.md via a natural-language prompt. The agent reads the existing plan and inserts the new content in the right location with consistent formatting.

- **Web shell** — `ta shell` now launches a browser-based shell by default. Features: live agent output streaming with auto-tail, `:tail <goal>` command to follow background runs, goal progress and status indicators, color-coded output, and scrollback. The TUI shell remains available via `ta shell --tui`.

- **Plugin Template Publication & Registry** — `ta plugin publish` uploads plugin templates to the community registry. `ta plugin search` discovers community plugins. Plugin installs now resolve from the registry with GitHub fallback.

- **Semantic Memory with RuVector** — Memory now uses the RuVector backend for similarity search. Relevant memories from past sessions are automatically retrieved and injected into agent context at goal start. Includes project constitution and plan-completion indexing.

- **Parallel Agent Sessions** — Multiple goals can now run concurrently. Each gets an isolated staging workspace. `ta shell` shows all active sessions and lets you switch between them.

- **VCS Adapter Externalization** — Perforce and SVN are now external plugins (JSON-over-stdio protocol) rather than built-in adapters. Git remains built-in. Custom VCS adapters can be installed in `~/.config/ta/plugins/vcs/` without rebuilding TA.

- **Template Projects & Bootstrap Flow** — `ta project new --template <name>` scaffolds from built-in project templates (web service, CLI tool, library). Templates include pre-configured PLAN.md, `.ta/` config, and agent guidance.

- **Event-Driven Agent Routing** — Daemon events (goal completion, draft ready, agent questions) can now trigger agent actions. Configurable routing rules in `daemon.toml`.

- **`ta build` (BuildAdapter)** — New `ta build` command wraps project build systems (cargo, npm, make) and captures output. Used by the release pipeline to verify builds before tagging.

### Improvements

- **Draft workflow**: `ta draft amend` lets you iterate on a draft without restarting the goal. Draft compositing chains parent and child drafts so follow-up goals build on approved changes. Transactional rollback restores source files if apply fails mid-way.

- **Shell paste-at-end** — Pasting multi-line content in the shell no longer submits on the first newline; it accumulates at the end of the input buffer and waits for an explicit submit.

- **Shell multi-agent UX** — Working indicator now clears correctly when an agent finishes. Auto-scroll triggers when the view is near the bottom. Multiple agent sessions visible simultaneously.

- **Discord notifications**: Cursor-based reconnect prevents duplicate notifications after daemon restart (v0.12.6). Recovery task state initialisation prevents re-emitting events for already-complete goals (v0.12.7 predecessor). Age filter (10-minute threshold) drops stale events as defense-in-depth (v0.12.8).

- **Agent output visibility** — Live streaming of agent stdout to the shell during goal execution. Background commands auto-tail. Agent thinking steps and tool calls visible in real-time with structured formatting.

- **Daemon watchdog** — Daemon monitors the agent subprocess PID. Orphaned agent processes are detected and cleaned up. Health check events suppressed from shell output.

- **Release pipeline robustness**: `find_latest_draft` now skips already-approved drafts (prevents re-approval errors). New `--from-tag` flag allows explicit control of the commit range for release note generation.

### Bug Fixes

- **`ta draft list` / `ta draft apply` silent failure** — `load_all_packages` now warns when a package file can't be read instead of silently skipping it. Fixes "draft not found" errors caused by binary version skew between daemon and CLI.

- **Discord plugin not rebuilt on install** — `install_local.sh` now explicitly builds and deploys the Discord channel plugin. Previously the plugin binary was never updated during local installs.

- **Release pipeline: cryptic error messages** — `ta draft apply` errors now show the goal title, plain-English explanation, and numbered steps to resolve. Pipeline wrappers include the exact `ta release run <version>` retry command.

- **Constitution compliance (§4)** — Injected CLAUDE.md and settings files are now correctly restored on all error paths in `ta run`. Follow-up goals restore from backup before re-injecting to prevent stale/nested content.

- **`.git/` leaking into staging diffs** — Staging overlay now excludes `.git/` from diffs so internal git metadata never appears as agent-authored changes.

- **Draft apply branch restore** — `ta draft apply` now correctly restores the original branch if the prepare step fails or if the target is a protected branch.
