## v0.7.5-alpha

### New Features

- **Channel Registry** ‚Äî Pluggable IO channel system where CLI, web, Slack, and webhook channels are all first-class citizens. New `ChannelFactory` trait with `build_review()` and `build_session()` methods, `ChannelRegistry` for type-based routing, and `.ta/config.yaml` channel routing configuration. Ships with built-in Terminal, AutoApprove, and Webhook factories. 10 tests.
- **API Mediator** ‚Äî Stage, preview, and approve intercepted MCP tool calls before they execute. New `ApiMediator` implements `ResourceMediator` for `mcp://` URIs with human-readable previews, risk flags (IRREVERSIBLE, EXTERNAL), and pattern-based classification (ReadOnly, Irreversible, ExternalSideEffect, StateChanging). 12 tests.
- **Agent-Guided Setup (`ta setup`)** ‚Äî Conversational setup flow where a TA agent helps configure your project. `ta setup wizard` auto-detects project type and generates a full `.ta/` config suite. `ta setup refine <section>` updates individual sections. `ta setup show` displays resolved config. All output is a reviewable TA draft. 5 tests.
- **Project Templates (`ta init`)** ‚Äî Bootstrap new TA-managed projects from templates. 5 built-in templates: `rust-workspace`, `typescript-monorepo`, `python-ml`, `go-service`, `generic`. Generates workflow.toml, memory.toml, policy.yaml, agent configs, .taignore, and constitutions. Parses Cargo.toml/package.json to seed `arch:module-map` memory entries. 10 tests.
- **`ta session close <id>`** ‚Äî New CLI command that safely closes interactive sessions. Automatically triggers `ta draft build` if the staging directory has uncommitted changes, preventing orphaned sessions when PTY exits abnormally. Supports `--no-draft` flag to skip draft build. 3 tests.

### Improvements

- **Memory backend toggle** ‚Äî `.ta/memory.toml` `backend` setting now respected: set `backend = "fs"` to skip RuVector and use filesystem-only memory, or `backend = "ruvector"` for semantic search.
- **Human guidance domain classification** ‚Äî Guidance events are now auto-classified into 7 project-aware domains using keyword matching. "Always use bun" stores as `conv:build-tool:slug` instead of a generic key. Explicit `domain:X` tags override auto-classification. 7 new tests.
- **PTY health check on resume** ‚Äî `ta session resume` now checks workspace health before reattaching. Reports clear recovery options (`ta session close` or `ta session abort`) if the workspace is missing or unhealthy.
- **Channel capability sets** ‚Äî Channels declare capabilities (supports_review, supports_session, supports_notify, supports_rich_media, supports_threads) so the registry can route requests to capable channels.

### Bug Fixes

- **Linux-musl cross-compilation fix** ‚Äî Platform-conditional `ioctl` type cast (`c_int` for musl, `c_ulong` for glibc/macOS) fixes build failure on `x86_64-unknown-linux-musl` and `aarch64-unknown-linux-musl` targets.
- **Release pipeline fail-as-one** ‚Äî Added `release-gate` job to GitHub Actions workflow. If any platform build fails, the entire release is blocked ‚Äî no more partial releases with missing platform binaries.
