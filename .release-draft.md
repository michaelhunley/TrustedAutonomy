## Public Beta â€” v0.13.17

This is the first public beta release of Trusted Autonomy. It represents the complete v0.13.x series: a local-first substrate for safe AI agent autonomy where every agent action is staged, reviewed, and explicitly approved before touching your real codebase.

**TA is invisible to the agent.** The agent works in an isolated staging copy using its normal tools. TA diffs the result, builds a structured draft, and presents it for review. You approve, deny, or selectively apply individual files. Nothing lands in your repo without your sign-off.

---

### Supervisor Agent

A second AI runs automatically after the main agent exits but before you review the draft. It checks changes against your goal and product constitution â€” flagging scope creep, unrelated edits, or constitution violations before they reach your review queue. Supports Claude Code (subscription or API key), Codex, and local Ollama models. Falls back gracefully to a warn verdict if unavailable â€” never blocks a draft build.

### VCS Environment Isolation

Spawned agents get a fully isolated git environment: their own `.git` initialised from the staging baseline, `GIT_CEILING_DIRECTORIES` blocks upward traversal. No more `index.lock` collisions between agent and developer processes. No accidental commits to your real repo from inside staging. Configurable modes: `isolated` (default), `inherit-read`, `none`.

### Gitignored Artifact Detection & Human Review Gate

TA-injected config files (`.mcp.json`, `settings.local.json`) are stripped from diffs before review. If gitignored build artifacts slip into a changeset, you get a human review gate rather than a silent `git add` abort.

### Goal Recovery & Watchdog

`ta goal recover` rescues goals stuck in `Finalizing` after a crash or daemon restart. Separate watchdog timers for stale (no heartbeat) vs. zombie (process gone, state not updated) goals. `heartbeat_required` lets long-running goals opt out of the stale timer without disabling zombie detection.

### VCS-Aware Team Setup & Large-Workspace Staging

`ta team setup` detects your project's VCS (git, Perforce, SVN) and configures team defaults. Large workspaces get smarter staging: configurable exclude patterns, size checks with actionable hints, and `.taignore` support so `node_modules` / `target` / build output never copy into staging.

### Product Constitution Framework

Define invariants your agents must never violate. `constitution.toml` describes architectural rules, prohibited patterns, and compliance requirements. The supervisor checks against them on every draft. Supports `extends` inheritance for team-wide base constitutions.

### Feature Velocity Stats

`ta stats velocity` â€” goals run, pass/fail rates, average draft build times, and phase completion velocity. `ta stats velocity-detail` for per-phase breakdown. `ta stats export` outputs CSV.

### Platform Installers

Native macOS DMG, Windows MSI (WiX v4), and Linux tarball. Release bundles include the Perforce VCS plugin and an offline copy of the usage guide (`USAGE.html`).

### Multi-Party Approval & Governance

`GovernanceConfig` â€” require N-of-M approvals before a draft can be applied. `ta draft approve --as <reviewer>` for named sign-offs. `--override` with full audit logging for emergency bypasses.

### Audit Attestation

Every audit event is signed with an Ed25519 key at write time. `ta audit verify-attestation` checks the full event log. Pluggable `AttestationBackend` trait for hardware key support.

### Sandbox Policy DSL

Declare what an agent is allowed to do before it runs â€” process allowlists, network rules, filesystem restrictions. Enforced via macOS `sandbox-exec` or Linux `bwrap`.

### Cross-Language Project Templates

`ta new` scaffolds Python, TypeScript, Go, and Rust projects with TA workflow config, `.taignore`, and constitution stubs included.

### Local Model Agent

`ta-agent-ollama` â€” run goals against a local Ollama model. Install with `ta agent install ollama`. No API key required.
