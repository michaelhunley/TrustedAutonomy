## v0.9.8-alpha

### New Features

- **Interactive TA Shell (`ta shell`)** — Full-screen TUI built with `ratatui` featuring scrolling output, input line, status bar, tab completion, and workflow mode. Classic line-mode shell available via `--classic` fallback.
- **Daemon API Expansion (`ta daemon`)** — The TA daemon is now a full HTTP API server with command execution, agent session management, SSE event streaming, project status, unified input routing, and bearer-token authentication. Any interface (terminal, web, Discord, Slack) can connect.
- **Pluggable Workflow Engine** — New `ta-workflow` crate with a `WorkflowEngine` trait, built-in YAML engine, process-based plugin bridge, verdict scoring, and interactive human-in-the-loop review. Full CLI/MCP/daemon/shell integration.
- **VCS Adapter Abstraction & Plugin Architecture** — Extended `SubmitAdapter` trait with `exclude_patterns()`, `save_state()`/`restore_state()`, `detect()`, and `revision_id()`. Added SVN and Perforce stub adapters, adapter auto-detection registry, and replaced hardcoded git calls with adapter-agnostic abstractions.
- **Orchestrator API & Goal-Scoped Agent Tracking** — `AgentSession` tracking in gateway state, `ta_agent_status` MCP tool, `ta status` CLI command, `AgentSessionStarted`/`AgentSessionEnded` events, and `CallerMode` enforcement for orchestrator operations.
- **Policy-Driven Auto-Approval** — Configurable draft auto-approval with path matching, policy conditions, and per-agent overrides. Drafts that meet policy criteria skip manual review.
- **Unified Allow/Deny List Pattern** — Reusable `AccessFilter` struct in `ta-policy` with consistent deny-takes-precedence semantics. Migrated daemon command routing, auto-approval path matching, and sandbox command policies.
- **Enhanced Draft View Output** — Module-grouped file listing, Design Decisions section showing alternatives considered, and `alternatives` parameter on the `ta_pr_build` MCP tool.
- **Event System & Subscription API** — Project-scoped event bus with `ta_event_subscribe` MCP tool, structured action envelopes for interface-agnostic rendering, and SSE streaming.
- **Solution Memory Export** — Local git-committed datastore for exporting and sharing solution patterns across sessions.
- **Interactive Developer Loop (`ta dev`)** — Streamlined inner-loop workflow for iterative development with live goal output streaming and `:tail` command in the shell.
- **Agent Framework Registry & Setup Integration** — Discover and configure supported agent frameworks (Claude Code, etc.) via `ta setup` with automatic detection and guided onboarding.
- **Extensibility Platform (v0.7.x)** — Channel Registry (Layer 5), API Mediator (Layer 1), agent-guided setup, project templates via `ta init`, and memory/config cleanup.
- **Platform Substrate (v0.6.x)** — Session & Human Control Plane (Layer 3), Unified Policy Config (Layer 2), Resource Mediation Trait (Layer 1), and Active Memory Injection with project-aware key schema.
- **Distribution & Packaging** — Cross-platform release binaries, native Windows support, and sandbox runner for optional hardening.
- **Goal History & Garbage Collection** — `ta goal gc` command with history ledger, `ta goal list --active` filtering, and `ta goal history` browsing.

### Improvements

- **Structured event actions** — Event envelopes now carry structured actions for interface-agnostic rendering across CLI, TUI, web, and chat channels.
- **Actionable event output** — `draft_built` and `goal_completed` events include next-step suggestions so users always know what to do.
- **Live goal output streaming** — Stream agent output in real time via the daemon and shell `:tail` command.
- **Periodic daemon health checks** — `ta shell` auto-reconnects to the daemon if the connection drops.
- **Unified crate versioning** — All workspace crates share a single version via `workspace.package.version`.
- **Configurable command routing** — Shell commands routed via `.ta/shell.toml` configuration.
- **Background long-running commands** — `ta run`/`ta dev` execute in the background with reliable daemon process management.
- **Pre-flight state validation** — `ta draft apply` validates goal state before applying changes.
- **Tiered diff explanations** — Contextual diff output adapted to the review channel.
- **USAGE.md restructured** — Quick Start rewritten around project setup, `ta dev`, and first goal workflow.
- **ADR: Product Concept Model** — Documented extension points and restructured roadmap around invisible mediation and conversational continuity.

### Bug Fixes

- **Goal state not transitioning on draft apply** — Fixed state machine so goals correctly advance when drafts are applied.
- **Auto-restore branch after draft apply** — Git branch is restored to its original position after applying a draft.
- **Agent subprocess wiring** — Fixed shell Q&A agent subprocess and restored goal output streaming.
- **SSE `since` filter** — Use `Z` suffix instead of unescaped `+` in URL timestamps.
- **`find_bin` prefers most recent binary** — Binary lookup now selects the most recently built artifact.
- **Daemon kill reliability** — Fixed process cleanup when stopping long-running daemon commands.
- **Daemon version detection** — Fixed version mismatch checks between CLI and daemon.
- **Command timeout for `ta run`/`ta dev`** — Increased timeouts so long-running agent sessions don't get killed prematurely.
- **Shell & daemon UX fixes** — Version checks, argument ordering, and phase-aware `ta run`.
- **Duplicate goal creation** — Added `--goal-id` flag to prevent creating duplicate goals in the orchestrator.
- **`duration_secs` timer placement** — Fixed timer so goal duration is measured accurately.
- **Disk fallback for MCP draft list** — Draft listing falls back to disk when in-memory state is unavailable.
- **Cursor-inclusive event polling** — Fixed off-by-one in event subscription cursor handling.
- **Orphaned draft cleanup** — `ta draft gc` now cleans up packages with no parent goal.
- **Event emission plumbing** — Wired `GoalStarted`, `GoalCompleted`, `GoalFailed`, and `DraftBuilt` events into all lifecycle paths (CLI and MCP).
- **`ta_goal_start` re-entrancy guard** — Uses `TA_IS_STAGING` env var instead of path sniffing to block nested staging.
- **Macro MCP injection** — Fixed MCP tool injection for macro/inner-loop goals.
- **PR "why" field** — Fixed missing motivation field in pull request descriptions.
- **`ta dev` exit handling** — Fixed clean exit from the interactive developer loop.
- **Strip `CLAUDECODE` env var on daemon startup** — Prevents credential leakage to child processes.
- **Windows release build** — Fixed compilation/linking issues preventing release builds on Windows targets.
